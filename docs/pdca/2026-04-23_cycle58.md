## 判定

### ⭐ LTC fragility は asset-specific

- **BTC**: 4y で -0.41%、DD 1.61% — ほぼフラット、2022 bear の影響軽微
- **ETH**: 4y で -10.17%、DD 17% — mild な損失
- **LTC**: 4y で -46.67%、DD 90.93% — **破壊的破綻**

同一 profile (v6) で同一期間なのに LTC だけが極端に破綻する = **LTC の 2022 価格ダイナミクスが profile の SL=14% stop-loss では捕らえきれないギャップ/連続下落を持つ**。BTC/ETH は同期間の bear でも max drawdown が 20% 以内に収まる。

### Regime routing の再活性化可能性

cycle40 で LTC 15m の regime detector が "常に bull-trend" を emit して dead と判定されたが、
**asset 側で regime を決めるなら復活**できる:
- BTC の DD / volatility を regime indicator として使う (BTC DD が大きい期間は LTC trading を抑制)
- あるいは BTC/LTC の相関を使って tail regime を検出

これは handoff_v2 cycle46+ 提案の "asset diversification" とも重なる。

### v6 promote への影響

v6 promote 根拠がさらに強化:
- **LTC 2022 fragility は profile 側で対処できない asset-specific 問題**
- つまり **v6 の "2022 で破綻する" 注意書きは LTC 固定運用の限界 that lies outside v6 scope**
- BTC/ETH への展開がある日実装されれば、v6 は 2022 でも生存できる可能性

## 学び

1. **Profile と asset の責任分離**: Profile は price action の典型パターンを捕らえる汎用ロジック。ここに asset-specific の fragility を押し込もうとすると local optimum に束縛される。**fragility は asset selection / regime router の層で対処すべき**であり、これは PR-5 regime routing / asset diversification (cycle40 撤退済み) の復活材料となる。
2. **BTC の 2022 耐性は大きな発見**: LTC での aggressive profile が BTC でも使えるなら、LTC live + BTC regime monitor の構成は production 強化候補。
3. **ETH は中間 asset**: BTC ほど耐性はないが、LTC ほど破綻しない。asset diversification で risk balance を取る素材として有望。
