## 学び (cycle49-52 統合)

### silent no-op / clamp / dead code の分類体系

本セッションで集まった **8 カテゴリの "値を変えても効かない" 現象**:

| 種別 | 例 | 検出方法 | 対応 |
|---|---|---|---|
| a. 完全 silent no-op | `htf_filter.alignment_boost` | 全セル bit-identical (trades) | コード修正 (live で使われていれば fix、backtest-only なら doc) |
| b. 階層 clamp (stance→signal) | `signal_rules.breakout.volume_ratio_min` (<= stance 側) | 下位値でセル bit-identical | 両軸同時 sweep |
| c. 階層 clamp (stance→contrarian) | `signal_rules.contrarian.rsi_entry` (>= stance) | 同上 | 同上 (cycle47 発見) |
| d. 機能 dead for TP-short | `trailing_atr_multiplier` (TP=4/5/6) | 全 trailing 値で bit-identical | TP を大きく広げる or 別 base |
| e. stance kill | `stance_rules.breakout_volume_ratio >= 2.1` | BREAKOUT 発火 0 で return plateau | stance 内の他値に戻す |

**検出パターンは共通**: **WFO の IS results で trades/return が完全一致するセルを探す**。
新 axis を追加するたびに silent check を dod に入れる習慣が必要。
